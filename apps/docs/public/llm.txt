# Banata Auth

> Project-scoped authentication infrastructure with dashboard-managed configuration, API keys, and app-side integration packages.

## Overview

Banata Auth is an authentication platform for products that want Banata to remain the system of record for auth. Teams create projects in the dashboard, configure authentication there or by code with the SDK, create project-scoped API keys, and connect their apps over HTTP.

## Primary Flow

1. Sign in to the Banata dashboard.
2. Create or open a project.
3. Configure auth methods, providers, emails, redirects, and branding.
4. Create a project-scoped API key.
5. Connect the app with `@banata-auth/nextjs`, `@banata-auth/react`, and `@banata-auth/sdk`.

## Core Concepts

- Projects are the isolation boundary.
- API keys determine project scope.
- Dashboard changes and SDK configuration writes update the same underlying project config.
- Self-hosting changes where Banata runs, not how customer apps integrate.
