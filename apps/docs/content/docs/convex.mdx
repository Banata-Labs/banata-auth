`@banata-auth/convex` is **not** the dashboard-first integration path. It is the self-hosting package for teams that want the auth engine to run inside their own Convex deployment.

If you want Banata to behave like WorkOS, start with the dashboard and [`@banata-auth/sdk`](/docs/sdk). Use this page only when you intentionally want self-hosting.

---

## What This Package Does

When you use `@banata-auth/convex`:

- users, sessions, organizations, and auth config live in your Convex deployment
- Better Auth is instantiated inside your Convex runtime
- you must register a local Convex component and generate a local adapter

That is why this package belongs in the **self-hosting** section of the docs.

---

## Supported Version Line

Use:

- `better-auth@1.4.21`
- `@convex-dev/better-auth@0.10.13`

The current package line should be treated as `better-auth 1.4.x` only.

## Package Entry Points

| Import | Use |
|---|---|
| `@banata-auth/convex` | Auth factory and route registration helpers |
| `@banata-auth/convex/schema` | Self-hosted schema re-export |
| `@banata-auth/convex/auth-config` | Convex JWT auth-config helper |
| `@banata-auth/convex/adapter` | Helper for generating local adapter CRUD functions |
| `@banata-auth/convex/node` | Node-side auth handling for the proxy route pattern |
| `@banata-auth/convex/plugins` | Advanced plugin access |

---

## Recommended Self-Hosted Flow

1. Register a local Convex component in `convex/banataAuth/convex.config.ts`.
2. Register that component in `convex/convex.config.ts`.
3. Re-export the schema in `convex/banataAuth/schema.ts`.
4. Build the auth factory in `convex/banataAuth/auth.ts`.
5. Generate the adapter CRUD functions in `convex/banataAuth/adapter.ts`.
6. Route requests through `convex/authNode.ts` and `convex/http.ts`.

The full file contents are in [Quick Start](/docs/quickstart).

---

## Why This Is Separate From The Public Story

The public product goal is dashboard-first. `@banata-auth/convex` exists for:

- Banata's own dogfooding
- GitHub-based self-hosting
- advanced teams that explicitly want to own the auth runtime

It should not be treated as the default first-run experience for most users.
