## Packages

Banata Auth ships as **5 npm packages** that you install in your application:

| Package | What It Does |
|---|---|
| **`@banata-auth/convex`** | The backend auth engine. Runs inside Convex вЂ” handles sessions, OAuth, database, and all auth logic. |
| **`@banata-auth/nextjs`** | Next.js integration. Provides the route handler (reverse proxy), middleware for route protection, and server-side auth utilities. |
| **`@banata-auth/react`** | React integration. Provider, hooks (`useUser`, `useSession`, `useOrganization`), and pre-built UI components. |
| **`@banata-auth/sdk`** | Server-side admin SDK. Programmatic management of users, organizations, SSO, webhooks, and more. |
| **`@banata-auth/shared`** | Shared types, validation schemas, error classes, and constants used across all packages. |

## Key Files Explained

| File | Purpose |
|---|---|
| `convex/banataAuth/auth.ts` | Your auth configuration вЂ” enable/disable auth methods, configure social providers, set up email callbacks |
| `convex/banataAuth/schema.ts` | Re-exports the Banata Auth database schema (26 tables) |
| `convex/auth.config.ts` | Tells Convex how to validate JWTs |
| `convex/http.ts` | Registers auth HTTP endpoints on your Convex deployment |
| `src/app/api/auth/[...all]/route.ts` | Reverse proxy вЂ” forwards auth requests to Convex, keeps cookies on your domain |
| `src/lib/auth-server.ts` | Server utilities вЂ” `isAuthenticated()`, `getToken()`, `preloadAuthQuery()` |
| `src/lib/auth-client.ts` | Client-side auth client for sign-in, sign-up, sign-out |
