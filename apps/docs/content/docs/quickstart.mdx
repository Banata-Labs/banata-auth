This guide walks you through integrating Banata Auth into an existing **Next.js + Convex** project. By the end, you'll have:

- Email/password authentication working
- Protected routes via middleware
- Auth state available in React components via hooks
- Server-side auth utilities for API routes and server components

---

## Prerequisites

Before you start, make sure you have:

- **Node.js 20+** or **Bun 1.2+**
- A **[Convex](https://convex.dev) account** with a project created
- An existing **Next.js project** with Convex already configured (if not, run `npx create-next-app@latest && npx convex dev --once` first)

You should already have these files in your project:

```
my-app/
├── convex/
│   ├── _generated/       # Auto-generated by Convex
│   └── ...
├── src/
│   └── app/
│       └── layout.tsx
├── .env.local            # CONVEX_DEPLOYMENT, NEXT_PUBLIC_CONVEX_URL
├── package.json
└── next.config.ts
```

## Step 1: Install Packages

Install the Banata Auth packages you need:

```bash
# Using npm
npm install @banata-auth/convex @banata-auth/nextjs @banata-auth/react @banata-auth/shared

# Using pnpm (monorepo)
pnpm add @banata-auth/convex @banata-auth/nextjs @banata-auth/react @banata-auth/shared

# Using Bun
bun add @banata-auth/convex @banata-auth/nextjs @banata-auth/react @banata-auth/shared
```

You also need these peer dependencies if you don't already have them:

```bash
npm install better-auth @convex-dev/better-auth convex
```

## Step 2: Set Environment Variables

Banata Auth requires environment variables in **two places**: your Next.js `.env.local` file and your Convex deployment.

### Next.js Environment (`.env.local`)

Add these to your `.env.local`:

```bash
# Already set by Convex
CONVEX_DEPLOYMENT=dev:your-deployment-name
NEXT_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud

# Add these for Banata Auth
NEXT_PUBLIC_CONVEX_SITE_URL=https://your-deployment.convex.site
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

> **How to find your Convex site URL:** Go to the [Convex dashboard](https://dashboard.convex.dev), select your project, and look for the "HTTP Actions" URL. It ends in `.convex.site`.

### Convex Environment Variables

Auth secrets must be set on the Convex deployment (not in `.env.local`), because they're used by server-side Convex functions:

```bash
# Generate a cryptographically secure secret
npx convex env set BETTER_AUTH_SECRET "$(openssl rand -base64 32)"

# Your app's public URL (used for OAuth callbacks, email links, etc.)
npx convex env set SITE_URL "http://localhost:3000"
```

> **Important:** The `BETTER_AUTH_SECRET` is used to sign sessions and tokens. It must be a strong, random string. Never commit it to source control. In production, use a different secret than development.

## Step 3: Create the Convex Auth Configuration

This is the core of your Banata Auth setup. You need three files in your `convex/` directory.

### 3a. Auth Config (`convex/auth.config.ts`)

This tells Convex how to validate JWTs issued by Better Auth:

```typescript
// convex/auth.config.ts
import { getAuthConfigProvider } from "@convex-dev/better-auth/auth-config";
import type { AuthConfig } from "convex/server";

export default {
  providers: [getAuthConfigProvider()],
} satisfies AuthConfig;
```

### 3b. Component Schema (`convex/banataAuth/schema.ts`)

Re-export the Banata Auth database schema. This defines the 26 tables used for auth data (users, sessions, accounts, organizations, etc.):

```typescript
// convex/banataAuth/schema.ts
export { default } from "@banata-auth/convex/schema";
```

### 3c. Auth Factory (`convex/banataAuth/auth.ts`)

This is where you configure Banata Auth — authentication methods, social providers, email callbacks, and more:

```typescript
// convex/banataAuth/auth.ts
import {
  createBanataAuth,
  createBanataAuthComponent,
  createBanataAuthOptions,
  type BanataAuthConfig,
} from "@banata-auth/convex";
import type { GenericCtx } from "@convex-dev/better-auth/utils";
import { components } from "../_generated/api";
import type { DataModel } from "../_generated/dataModel";
import authConfig from "../auth.config";
import schema from "./schema";

// Create the auth component (connects to Convex's component system)
export const authComponent = createBanataAuthComponent<
  DataModel,
  typeof schema
>(components.banataAuth, schema);

// Define your auth configuration
function buildConfig(): BanataAuthConfig {
  return {
    appName: "My App",
    siteUrl: process.env.SITE_URL ?? "http://localhost:3000",
    secret: (() => {
      const secret = process.env.BETTER_AUTH_SECRET;
      if (!secret) {
        throw new Error(
          "BETTER_AUTH_SECRET environment variable is required. " +
          "Set it via: npx convex env set BETTER_AUTH_SECRET <your-secret>"
        );
      }
      return secret;
    })(),

    // Enable the auth methods you need
    authMethods: {
      emailPassword: true,    // Email & password sign-up/sign-in
      // magicLink: true,      // Passwordless email links
      // emailOtp: true,       // One-time password via email
      // twoFactor: true,      // TOTP-based MFA
      // passkey: true,        // WebAuthn biometric/hardware keys
      // anonymous: true,      // Anonymous sessions
      // multiSession: true,   // Multiple concurrent sessions
      // organization: true,   // Multi-tenant organizations
    },

    // Email callbacks (required for email-based auth methods)
    email: {
      sendVerificationEmail: async ({ user, url }) => {
        console.log(`[DEV] Verification email for ${user.email}: ${url}`);
        // In production, send via Resend, SendGrid, etc.
      },
      sendResetPassword: async ({ user, url }) => {
        console.log(`[DEV] Password reset for ${user.email}: ${url}`);
      },
      // sendMagicLink: async ({ email, url }) => { ... },
      // sendOtp: async ({ email, otp }) => { ... },
      // sendInvitationEmail: async ({ invitation }) => { ... },
    },

    // Social OAuth providers (optional)
    // socialProviders: {
    //   github: {
    //     clientId: process.env.GITHUB_CLIENT_ID!,
    //     clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    //   },
    //   google: {
    //     clientId: process.env.GOOGLE_CLIENT_ID!,
    //     clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
    //   },
    // },
  };
}

// Auth options factory (used for type inference)
export const createAuthOptions = (ctx: GenericCtx<DataModel>) =>
  createBanataAuthOptions(ctx, {
    authComponent,
    authConfig,
    config: buildConfig(),
  });

// Auth factory (used in HTTP routes and Convex functions)
export const createAuth = (ctx: GenericCtx<DataModel>) =>
  createBanataAuth(ctx, {
    authComponent,
    authConfig,
    config: buildConfig(),
  });
```

## Step 4: Register HTTP Routes

Better Auth needs HTTP endpoints to handle sign-in, sign-up, OAuth callbacks, JWKS, and other auth flows. Register them in your Convex HTTP router:

```typescript
// convex/http.ts
import { httpRouter } from "convex/server";
import { authComponent, createAuth } from "./banataAuth/auth";

const http = httpRouter();

// Register all Better Auth routes:
//   /api/auth/sign-in, /api/auth/sign-up, /api/auth/callback/*,
//   /api/auth/.well-known/jwks.json, and more
authComponent.registerRoutes(http, createAuth);

export default http;
```

## Step 5: Create the Next.js Route Handler

The route handler acts as a **reverse proxy** — it forwards auth requests from your Next.js app to the Convex `.site` URL where Better Auth is running. This keeps auth cookies on your domain.

### 5a. Server Auth Utilities (`src/lib/auth-server.ts`)

```typescript
// src/lib/auth-server.ts
import { convexBetterAuthNextJs } from "@convex-dev/better-auth/nextjs";

export const {
  handler,            // Route handler proxy (GET, POST)
  isAuthenticated,    // Check auth in server components
  getToken,           // Get JWT for Convex client
  preloadAuthQuery,   // Preload authenticated queries in RSC
  fetchAuthQuery,     // Fetch authenticated query results
  fetchAuthMutation,  // Execute authenticated mutations
  fetchAuthAction,    // Execute authenticated actions
} = convexBetterAuthNextJs({
  convexUrl: process.env.NEXT_PUBLIC_CONVEX_URL!,
  convexSiteUrl: process.env.NEXT_PUBLIC_CONVEX_SITE_URL!,
});
```

### 5b. API Route Handler (`src/app/api/auth/[...all]/route.ts`)

```typescript
// src/app/api/auth/[...all]/route.ts
import { handler } from "@/lib/auth-server";

export const { GET, POST } = handler;
```

> **Note:** The catch-all route `[...all]` matches all paths under `/api/auth/`, forwarding requests like `/api/auth/sign-in/email`, `/api/auth/sign-up/email`, `/api/auth/callback/github`, etc. to Convex.

## Step 6: Create the Auth Client

The client-side auth client is used by React components to call auth endpoints (sign in, sign up, sign out, etc.):

```typescript
// src/lib/auth-client.ts
import { createAuthClient } from "better-auth/react";

const baseURL =
  typeof window !== "undefined"
    ? "/api/auth"
    : `${process.env.NEXT_PUBLIC_SITE_URL ?? "http://localhost:3000"}/api/auth`;

export const authClient = createAuthClient({
  baseURL,
});
```

The `baseURL` points to your Next.js route handler. On the client it uses a relative URL (`/api/auth`); on the server it uses the absolute URL since there's no browser context.

## Step 7: Add the React Provider

Wrap your app in the auth provider so hooks work throughout your component tree:

```tsx
// src/app/layout.tsx
import { ConvexClientProvider } from "@/components/convex-client-provider";

export const metadata = {
  title: "My App",
  description: "Powered by Banata Auth",
};

export default async function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <ConvexClientProvider>
          {children}
        </ConvexClientProvider>
      </body>
    </html>
  );
}
```

Create the Convex client provider:

```tsx
// src/components/convex-client-provider.tsx
"use client";

import { ConvexProvider, ConvexReactClient } from "convex/react";
import { BanataConvexProvider } from "@banata-auth/react/convex";

const convex = new ConvexReactClient(
  process.env.NEXT_PUBLIC_CONVEX_URL!
);

export function ConvexClientProvider({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <ConvexProvider client={convex}>
      <BanataConvexProvider>
        {children}
      </BanataConvexProvider>
    </ConvexProvider>
  );
}
```

> **`BanataConvexProvider`** handles JWT token exchange — it fetches a Convex-compatible token from the auth endpoint and passes it to the Convex client so your authenticated queries and mutations work.

## Step 8: Use Auth in Your Components

### Sign Up Form

```tsx
// src/app/sign-up/page.tsx
"use client";

import { authClient } from "@/lib/auth-client";
import { useState } from "react";

export default function SignUpPage() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [name, setName] = useState("");
  const [error, setError] = useState("");

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    setError("");

    const { error } = await authClient.signUp.email({
      email,
      password,
      name,
    });

    if (error) {
      setError(error.message ?? "Sign up failed");
      return;
    }

    // Redirect to app on success
    window.location.href = "/dashboard";
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        placeholder="Name"
        value={name}
        onChange={(e) => setName(e.target.value)}
        required
      />
      <input
        type="email"
        placeholder="Email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        required
      />
      <input
        type="password"
        placeholder="Password (8+ characters)"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        minLength={8}
        required
      />
      {error && <p style={{ color: "red" }}>{error}</p>}
      <button type="submit">Sign Up</button>
    </form>
  );
}
```

### Using Auth State with Hooks

```tsx
// src/app/dashboard/page.tsx
"use client";

import { useUser, useSession } from "@banata-auth/react";
import { authClient } from "@/lib/auth-client";

export default function DashboardPage() {
  const { user, isLoading } = useUser();
  const { session } = useSession();

  if (isLoading) {
    return <div>Loading...</div>;
  }

  if (!user) {
    return (
      <div>
        <p>You are not signed in.</p>
        <a href="/sign-in">Sign In</a>
      </div>
    );
  }

  return (
    <div>
      <h1>Welcome, {user.name}!</h1>
      <p>Email: {user.email}</p>
      <p>Session ID: {session?.id}</p>
      <button onClick={() => authClient.signOut()}>
        Sign Out
      </button>
    </div>
  );
}
```

### Server-Side Auth Check

```tsx
// src/app/dashboard/layout.tsx
import { isAuthenticated } from "@/lib/auth-server";
import { redirect } from "next/navigation";

export default async function DashboardLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const authed = await isAuthenticated();

  if (!authed) {
    redirect("/sign-in");
  }

  return <>{children}</>;
}
```

## Troubleshooting

### "BETTER_AUTH_SECRET environment variable is required"

You haven't set the secret on your Convex deployment:

```bash
npx convex env set BETTER_AUTH_SECRET "$(openssl rand -base64 32)"
```

### "Failed to fetch" or Network Errors on Sign In/Sign Up

1. **Check that Convex dev server is running** — You need `npx convex dev` running alongside your Next.js dev server.
2. **Check `NEXT_PUBLIC_CONVEX_SITE_URL`** — This must point to your Convex HTTP actions URL (ending in `.convex.site` for cloud, or `http://127.0.0.1:3211` for local).
3. **Check the route handler exists** — `src/app/api/auth/[...all]/route.ts` must export `GET` and `POST`.

### OAuth Callback Errors

1. **Callback URL must match** — In your OAuth provider's settings (e.g., GitHub), set the callback URL to `http://localhost:3000/api/auth/callback/github` (or whichever provider).
2. **Set provider credentials on Convex** — OAuth secrets go on the Convex deployment:
   ```bash
   npx convex env set GITHUB_CLIENT_ID "your-client-id"
   npx convex env set GITHUB_CLIENT_SECRET "your-client-secret"
   ```

### Hooks Return `undefined` User

1. **Make sure `BanataConvexProvider` wraps your app** — Check that your layout includes the provider from Step 7.
2. **Wait for loading** — Always check `isLoading` before reading `user`:
   ```tsx
   const { user, isLoading } = useUser();
   if (isLoading) return <div>Loading...</div>;
   ```

### "Module not found: @banata-auth/convex"

If you're in a monorepo, make sure the packages are properly linked. With pnpm workspaces:

```bash
pnpm install
```

With npm workspaces, add the packages to your root `package.json`:

```json
{
  "workspaces": ["apps/*", "packages/*"]
}
```

### Session Cookie Not Being Set

1. **Same-origin requirement** — The auth route handler must be on the same domain as your app. Don't call the Convex `.site` URL directly from the browser.
2. **HTTPS in production** — Cookies with `Secure` flag require HTTPS. Use `http://localhost` in development, `https://your-domain.com` in production.
