This guide is the **self-hosted** setup for Banata Auth.

If you want a **dashboard-first / WorkOS-style** integration, start with the dashboard and [`@banata-auth/sdk`](/docs/sdk) instead. This guide is specifically for running the auth engine inside your own Convex deployment.

---

## Before You Start

Use these versions for the current Convex integration:

```bash
better-auth@1.4.21
@convex-dev/better-auth@0.10.13
convex@^1.25.0
```

> `@banata-auth/*@0.1.0` was published with broken `workspace:*` internal dependencies. If you are installing the published `0.1.0` build today, add temporary overrides until the patched release is available:

```json
{
  "overrides": {
    "@banata-auth/shared": "0.1.0",
    "@banata-auth/react": "0.1.0"
  }
}
```

Install the packages:

```bash
bun add better-auth@1.4.21 @convex-dev/better-auth@0.10.13 convex @banata-auth/convex @banata-auth/nextjs @banata-auth/react @banata-auth/shared
```

## Required Backend Files

The self-hosted Convex setup needs **all** of these files:

```text
convex/
├── convex.config.ts
├── auth.config.ts
├── authNode.ts
├── http.ts
└── banataAuth/
    ├── convex.config.ts
    ├── schema.ts
    ├── auth.ts
    └── adapter.ts
```

### 1. `convex/banataAuth/convex.config.ts`

```ts
import { defineComponent } from "convex/server";

const component = defineComponent("banataAuth");

export default component;
```

### 2. `convex/convex.config.ts`

```ts
import { defineApp } from "convex/server";
import banataAuth from "./banataAuth/convex.config";

const app = defineApp();

app.use(banataAuth);

export default app;
```

### 3. `convex/auth.config.ts`

```ts
import { getAuthConfigProvider } from "@banata-auth/convex/auth-config";
import type { AuthConfig } from "convex/server";

export default {
  providers: [getAuthConfigProvider()],
} satisfies AuthConfig;
```

### 4. `convex/banataAuth/schema.ts`

```ts
export { default } from "@banata-auth/convex/schema";
```

### 5. `convex/banataAuth/auth.ts`

```ts
import {
  type BanataAuthConfig,
  createBanataAuth,
  createBanataAuthComponent,
  createBanataAuthOptions,
} from "@banata-auth/convex";
import type { GenericCtx } from "@convex-dev/better-auth/utils";
import { components } from "../_generated/api";
import type { DataModel } from "../_generated/dataModel";
import authConfig from "../auth.config";
import schema from "./schema";

export const authComponent = createBanataAuthComponent<DataModel, typeof schema>(
  components.banataAuth,
  schema,
);

export function buildConfig(): BanataAuthConfig {
  const secret = process.env.BETTER_AUTH_SECRET;
  if (!secret) {
    throw new Error(
      "BETTER_AUTH_SECRET is required. Set it with: npx convex env set BETTER_AUTH_SECRET <value>",
    );
  }

  return {
    appName: "My App",
    siteUrl: process.env.SITE_URL ?? "http://localhost:3000",
    secret,
    authMethods: {
      emailPassword: true,
      organization: true,
    },
    email: {
      sendVerificationEmail: async ({ user, url }) => {
        console.log(`[DEV] verification email for ${user.email}: ${url}`);
      },
      sendResetPassword: async ({ user, url }) => {
        console.log(`[DEV] reset password email for ${user.email}: ${url}`);
      },
    },
  };
}

export const createAuthOptions = (ctx: GenericCtx<DataModel>) =>
  createBanataAuthOptions(ctx, {
    authComponent,
    authConfig,
    config: buildConfig(),
  });

export const createAuth = (ctx: GenericCtx<DataModel>) =>
  createBanataAuth(ctx, {
    authComponent,
    authConfig,
    config: buildConfig(),
  });
```

### 6. `convex/banataAuth/adapter.ts`

This file is required. Without it, auth operations fail with unresolved adapter function errors.

```ts
import { createApi } from "@banata-auth/convex/adapter";
import { createAuthOptions } from "./auth";
import schema from "./schema";

export const { create, findOne, findMany, updateOne, updateMany, deleteOne, deleteMany } =
  createApi(schema, createAuthOptions);
```

### 7. `convex/authNode.ts`

This matches the reference app and keeps auth requests routed through a Node action when Banata needs Node-only libraries.

```ts
"use node";

import { createBanataNodeAuth, handleBanataNodeAuthRequest } from "@banata-auth/convex/node";
import type { GenericCtx } from "@convex-dev/better-auth/utils";
import { v } from "convex/values";
import type { DataModel } from "./_generated/dataModel";
import { internalAction } from "./_generated/server";
import authConfig from "./auth.config";
import { authComponent, buildConfig } from "./banataAuth/auth";

const serializedHeader = v.object({
  key: v.string(),
  value: v.string(),
});

const createNodeAuth = (ctx: GenericCtx<DataModel>) =>
  createBanataNodeAuth(ctx, {
    authComponent,
    authConfig,
    config: buildConfig(),
  });

export const handleAuthRequest = internalAction({
  args: {
    request: v.object({
      method: v.string(),
      url: v.string(),
      headers: v.array(serializedHeader),
      body: v.union(v.string(), v.null()),
    }),
  },
  returns: v.object({
    status: v.number(),
    headers: v.array(serializedHeader),
    body: v.union(v.string(), v.null()),
  }),
  handler: async (ctx, args) => {
    return await handleBanataNodeAuthRequest(ctx, createNodeAuth, args.request);
  },
});
```

### 8. `convex/http.ts`

```ts
import { registerBanataAuthNodeProxyRoutes } from "@banata-auth/convex";
import { httpRouter } from "convex/server";
import { internal } from "./_generated/api";
import { createAuth } from "./banataAuth/auth";

const http = httpRouter();

registerBanataAuthNodeProxyRoutes(http, createAuth, internal.authNode.handleAuthRequest);

export default http;
```

## Environment Variables

Set the required secrets on the Convex deployment:

```bash
npx convex env set BETTER_AUTH_SECRET "<strong-random-secret>"
npx convex env set SITE_URL "http://localhost:3000"
```

For Next.js, keep your local app variables in `.env.local`:

```bash
CONVEX_DEPLOYMENT=dev:your-deployment
NEXT_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud
NEXT_PUBLIC_CONVEX_SITE_URL=https://your-deployment.convex.site
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

## Next.js Wiring

### `src/lib/auth-server.ts`

Use Banata's wrapper, not `@convex-dev/better-auth` directly:

```ts
import { createBanataAuthServer } from "@banata-auth/nextjs/server";

export const {
  handler,
  isAuthenticated,
  getToken,
  preloadAuthQuery,
  fetchAuthQuery,
  fetchAuthMutation,
  fetchAuthAction,
} = createBanataAuthServer({
  convexUrl: process.env.NEXT_PUBLIC_CONVEX_URL!,
  convexSiteUrl: process.env.NEXT_PUBLIC_CONVEX_SITE_URL!,
});
```

### `src/app/api/auth/[...all]/route.ts`

```ts
import { handler } from "@/lib/auth-server";

export const { GET, POST } = handler;
```

### `src/lib/auth-client.ts`

```ts
import { createAuthClient } from "@banata-auth/react/plugins";

const baseURL =
  typeof window !== "undefined"
    ? "/api/auth"
    : `${process.env.NEXT_PUBLIC_SITE_URL ?? "http://localhost:3000"}/api/auth`;

export const authClient = createAuthClient({
  baseURL,
});
```

At this point you can build sign-in and sign-up pages with `authClient`, or continue with the React guide to add `BanataAuthProvider`, hooks, and prebuilt auth components.

## Troubleshooting

### Install fails with `workspace:*` dependency errors

You are almost certainly on published `@banata-auth/*@0.1.0`. Use the temporary `overrides` block shown above until the patched release is published.

### `Couldn't resolve banataAuth.adapter.findOne`

You are missing one of these files:

- `convex/banataAuth/adapter.ts`
- `convex/banataAuth/convex.config.ts`
- `convex/convex.config.ts`

### Build breaks after installing `better-auth@latest`

Use `better-auth@1.4.21`. The current Convex integration is not validated against the `1.5.x` line.

### `convex.config.ts` is at the project root

For this setup, the app config belongs at `convex/convex.config.ts`.
