Hosted sign-in is available, but it is not the primary Banata integration path right now.

The main docs path focuses on:

- configuring projects from the dashboard
- binding apps with project-scoped API keys
- rendering auth in your own app with `@banata-auth/nextjs` and `@banata-auth/react`

Use hosted sign-in only when you specifically want Banata to render the primary auth surface.

---

## How It Works

In the hosted model:

1. your app links to Banata's hosted UI
2. Banata authenticates the user on the Banata domain
3. Banata redirects back to your app
4. your app finalizes the session on its own domain

---

## What Must Be Configured First

Hosted UI only reflects the configuration already saved for that project.

Before you test it, make sure the project has:

1. at least one enabled auth method
2. provider credentials for the methods you expect to render
3. email delivery configured for email-based flows
4. branding configured if you want the hosted pages to match the project

---

## Customer App Requirements

Your app still needs:

1. `/api/auth/[...all]` proxy route
2. a valid project-scoped API key on the server
3. the callback handling required by the hosted flow

If your app is missing those pieces, hosted auth may succeed on Banata but still fail to create a usable session in the app.

---

## Branding And Configuration

Hosted UI uses the same project configuration the dashboard and SDK use.

That means:

- dashboard branding changes update the hosted UI
- SDK configuration writes update the hosted UI
- there is no separate hosted-auth theme file to maintain

## Troubleshooting

### The hosted page loads but shows no buttons or form

The project is missing dashboard configuration. Check **Authentication -> Methods** and **Authentication -> Providers**.

### The hosted flow finishes but the user is still unauthenticated in the app

Check:

1. `/auth/callback` exists
2. `OPTIONS` is exported from your auth route
3. your auth route allows the hosted UI origin
4. your app uses a real project-scoped API key

### The hosted page says `Rate limit exceeded`

Banata rate-limits sensitive auth endpoints per project, IP, path, and, for email-based flows, the submitted identifier.

If you are repeatedly testing from one shared IP:

1. wait for the retry window to expire
2. try again with a fresh email if you are testing sign-up
3. avoid hammering the same hosted auth form while you are iterating

### The hosted page loads but returns the wrong project

Check that `client_id` is the project's client ID from the Banata dashboard, not the workspace slug or workspace name.
