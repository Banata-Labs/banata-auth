## What Banata Is

Banata Auth is an authentication platform for products that need:

- users and sessions
- organizations
- custom roles and permissions
- SSO and SCIM
- audit logs and webhooks
- email delivery and templates
- project-scoped API keys

Banata is the system of record. Your app does not store the primary auth data by default. Your app connects to Banata over HTTP, and Banata owns the auth runtime, configuration, and operational data.

---

## The Product Contract

Banata always starts the same way:

1. Sign in to the Banata dashboard.
2. Create or open a project.
3. Configure auth methods, providers, emails, redirects, and branding for that project.
4. Create a project-scoped API key.
5. Bind your app to that project with the API key.
6. Start auth flows from your app through Banata's route proxy, SDKs, and UI helpers.

If you skip the dashboard and API key bootstrap, Banata should not become a usable auth runtime for that app.

---

## Hosted Vs Self-Hosted

These are deployment choices, not different integration models.

- **Hosted Banata**: you use Banata's hosted dashboard and API.
- **Self-hosted Banata**: you deploy the Banata platform yourself, but your developers still sign in to a Banata dashboard, create projects, create API keys, and use the same SDKs.

Self-hosting changes where Banata runs. It does not change how customer apps connect to it.

## Package Roles

| Package | Role |
|---|---|
| `@banata-auth/nextjs` | App-side auth proxy and server helpers |
| `@banata-auth/react` | App-side UI, provider, and auth hooks |
| `@banata-auth/sdk` | Programmatic project configuration and admin APIs |
| `@banata-auth/convex` | Platform runtime package for Banata operators |
| `@banata-auth/shared` | Shared types, errors, and validation |

---

## What Matters First

- **Projects** are the isolation boundary.
- **API keys** are how your app binds to a project.
- **Dashboard config and SDK config are the same underlying config.**
- **Your app does not own the auth database. Banata does.**
- **Self-hosting changes infrastructure, not the customer integration model.**
