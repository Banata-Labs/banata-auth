The `@banata-auth/nextjs` package provides everything you need to integrate Banata Auth with a Next.js application — a reverse proxy route handler, route protection middleware, server-side auth utilities, and client hooks for preloaded queries.

```bash
npm install @banata-auth/nextjs
```

## Route Handler (Reverse Proxy)

The route handler acts as a **reverse proxy** — it forwards auth requests from your Next.js app to the Convex `.site` URL where Better Auth is running. This keeps session cookies on your domain.

### `createRouteHandler(options)`

```typescript
// src/app/api/auth/[...all]/route.ts
import { createRouteHandler } from "@banata-auth/nextjs";

export const { GET, POST } = createRouteHandler({
  // Required: The Convex site URL where HTTP actions are hosted
  convexSiteUrl: process.env.NEXT_PUBLIC_CONVEX_SITE_URL!,
});
```

Or use the `convexBetterAuthNextJs` factory (from `@convex-dev/better-auth`) which provides both the handler and server utilities:

```typescript
// src/lib/auth-server.ts
import { convexBetterAuthNextJs } from "@convex-dev/better-auth/nextjs";

export const {
  handler,
  isAuthenticated,
  getToken,
  preloadAuthQuery,
  fetchAuthQuery,
  fetchAuthMutation,
  fetchAuthAction,
} = convexBetterAuthNextJs({
  convexUrl: process.env.NEXT_PUBLIC_CONVEX_URL!,
  convexSiteUrl: process.env.NEXT_PUBLIC_CONVEX_SITE_URL!,
});

// src/app/api/auth/[...all]/route.ts
import { handler } from "@/lib/auth-server";
export const { GET, POST } = handler;
```

### How the Proxy Works

1. Browser sends request to `/api/auth/sign-in/email`
2. Route handler receives the request
3. Forwards to `{CONVEX_SITE_URL}/api/auth/sign-in/email`
4. Preserves all headers and body
5. Overrides the `host` header to match the Convex site
6. Uses `redirect: "manual"` for OAuth flows
7. Returns the response (including `Set-Cookie` headers)

**All 5 HTTP methods** are forwarded: GET, POST, PUT, PATCH, DELETE.

On network errors (Convex is unreachable), a **502 Bad Gateway** response is returned.

## Middleware (Route Protection)

The middleware checks for the Better Auth session cookie on incoming requests and redirects unauthenticated users.

### `banataAuthProxy(options)` / `banataAuthMiddleware(options)`

These are aliases for the same function:

```typescript
// middleware.ts
import { banataAuthProxy } from "@banata-auth/nextjs/middleware";

export default banataAuthProxy({
  // Routes that don't require authentication
  publicRoutes: [
    "/",
    "/sign-in",
    "/sign-up",
    "/forgot-password",
    "/api/webhooks/(.*)",  // Webhook endpoints
  ],

  // Where to redirect unauthenticated users
  signInUrl: "/sign-in",  // Default: "/sign-in"
});

export const config = {
  // Match all routes except Next.js internals and static files
  matcher: ["/((?!_next|static|favicon.ico|.*\\..*).*)"],
};
```

### Route Matching

Routes are matched using **regex with `^...$` anchoring**. This means:

| Pattern | Matches | Doesn't Match |
|---|---|---|
| `/dashboard` | `/dashboard` only | `/dashboard/settings` |
| `/dashboard(.*)` | `/dashboard`, `/dashboard/settings`, `/dashboard/users/123` | `/other` |
| `/api/webhooks/(.*)` | `/api/webhooks/banata`, `/api/webhooks/stripe` | `/api/webhooks` |
| `/sign-(in\|up)` | `/sign-in`, `/sign-up` | `/sign-out` |

### How Middleware Works

1. Request comes in
2. Check if the route matches any `publicRoutes` → Allow through
3. Check for Better Auth session cookie (via `getSessionCookie()`)
4. **Cookie found** → Set `x-pathname` header and allow through
5. **No cookie** → Redirect to `signInUrl` with `?redirect_url={original_path}`

### The `x-pathname` Header

The middleware sets an `x-pathname` header on authenticated requests. You can use this in your server components to know the current path:

```typescript
import { headers } from "next/headers";

export default async function Layout() {
  const headersList = await headers();
  const pathname = headersList.get("x-pathname");
  // ...
}
```

## Server Utilities

### `createBanataAuthServer(options)`

A factory that returns server-side auth utilities:

```typescript
// src/lib/auth-server.ts
import { createBanataAuthServer } from "@banata-auth/nextjs/server";

export const {
  handler,            // Route handler for /api/auth/[...all]
  isAuthenticated,    // Check if current request is authenticated
  getToken,           // Get JWT token for Convex
  preloadAuthQuery,   // Preload authenticated Convex queries
  fetchAuthQuery,     // Fetch authenticated query results
  fetchAuthMutation,  // Execute authenticated mutations
  fetchAuthAction,    // Execute authenticated actions
} = createBanataAuthServer({
  convexUrl: process.env.NEXT_PUBLIC_CONVEX_URL!,
  convexSiteUrl: process.env.NEXT_PUBLIC_CONVEX_SITE_URL!,
});
```

> **Note:** In practice, the example app uses `convexBetterAuthNextJs()` from `@convex-dev/better-auth/nextjs` which provides the same interface. Both approaches work.

### `isAuthenticated()`

Check if the current request has a valid session. Use in server components and server actions:

```tsx
// Server component
import { isAuthenticated } from "@/lib/auth-server";
import { redirect } from "next/navigation";

export default async function ProtectedPage() {
  const authed = await isAuthenticated();

  if (!authed) {
    redirect("/sign-in");
  }

  return <div>You are authenticated</div>;
}
```

### `getToken()`

Get a Convex-compatible JWT token for making authenticated queries:

```typescript
import { getToken } from "@/lib/auth-server";

export default async function ServerComponent() {
  const token = await getToken();

  if (token) {
    // Use token with Convex client
    // const result = await fetchQuery(api.myQuery, {}, { token });
  }
}
```

**Token flow:**
1. Reads session cookies from the request
2. Calls `/api/auth/convex/token` on the Convex `.site` URL
3. Returns the JWT token (or null if not authenticated)

### `preloadAuthQuery(query, args)`

Preload an authenticated Convex query in a server component, then consume it client-side without waterfalls:

```tsx
// Server component (page.tsx)
import { preloadAuthQuery } from "@/lib/auth-server";
import { api } from "@/convex/_generated/api";
import { DashboardClient } from "./dashboard-client";

export default async function DashboardPage() {
  const preloaded = await preloadAuthQuery(api.dashboard.getData, {});

  return <DashboardClient preloadedData={preloaded} />;
}
```

```tsx
// Client component (dashboard-client.tsx)
"use client";

import { usePreloadedAuthQuery } from "@banata-auth/nextjs/client";

export function DashboardClient({ preloadedData }) {
  const data = usePreloadedAuthQuery(preloadedData);

  return <div>{JSON.stringify(data)}</div>;
}
```

### `fetchAuthQuery(query, args)` / `fetchAuthMutation(mutation, args)` / `fetchAuthAction(action, args)`

Execute authenticated Convex operations from server components and server actions:

```tsx
import { fetchAuthQuery, fetchAuthMutation } from "@/lib/auth-server";
import { api } from "@/convex/_generated/api";

// In a server component
export default async function SettingsPage() {
  const settings = await fetchAuthQuery(api.settings.get, {});
  return <SettingsForm settings={settings} />;
}

// In a server action
export async function updateSettings(formData: FormData) {
  "use server";
  await fetchAuthMutation(api.settings.update, {
    name: formData.get("name") as string,
  });
}
```

## Client Hook

### `usePreloadedAuthQuery(preloadedQuery)`

Consume a preloaded auth query on the client side:

```tsx
"use client";

import { usePreloadedAuthQuery } from "@banata-auth/nextjs/client";
import type { Preloaded } from "convex/react";
import type { api } from "@/convex/_generated/api";

export function DataView({
  preloaded,
}: {
  preloaded: Preloaded<typeof api.myQuery>;
}) {
  const data = usePreloadedAuthQuery(preloaded);

  return <div>{data.name}</div>;
}
```

This hook:
1. Uses the preloaded data for the initial render (no loading state)
2. Subscribes to real-time updates via Convex
3. Re-renders automatically when data changes

## Two Route Handler Approaches

There are two ways to set up the auth route handler:

### Approach 1: `createRouteHandler` (from `@banata-auth/nextjs`)

Simple proxy — just forwards requests:

```typescript
// src/app/api/auth/[...all]/route.ts
import { createRouteHandler } from "@banata-auth/nextjs";
export const { GET, POST } = createRouteHandler({
  convexSiteUrl: process.env.NEXT_PUBLIC_CONVEX_SITE_URL!,
});
```

**Use when:** You only need the proxy and handle server auth separately.

### Approach 2: `convexBetterAuthNextJs` (from `@convex-dev/better-auth`)

Full-featured — provides the handler plus server utilities:

```typescript
// src/lib/auth-server.ts
import { convexBetterAuthNextJs } from "@convex-dev/better-auth/nextjs";

export const {
  handler,
  isAuthenticated,
  getToken,
  preloadAuthQuery,
  fetchAuthQuery,
  fetchAuthMutation,
  fetchAuthAction,
} = convexBetterAuthNextJs({
  convexUrl: process.env.NEXT_PUBLIC_CONVEX_URL!,
  convexSiteUrl: process.env.NEXT_PUBLIC_CONVEX_SITE_URL!,
});

// src/app/api/auth/[...all]/route.ts
import { handler } from "@/lib/auth-server";
export const { GET, POST } = handler;
```

**Use when:** You need server-side auth utilities (recommended for most apps).
