`@banata-auth/react` is the main UI package for customer apps.

Use it with [`@banata-auth/nextjs`](/docs/nextjs) when you want your app to own the auth screens while Banata owns the backend runtime and data.

```bash
npm install @banata-auth/react
```

---

## What It Provides

| Import | What It Provides |
|---|---|
| `@banata-auth/react` | UI components, provider, hooks |
| `@banata-auth/react/plugins` | `createAuthClient` and client plugins |
| `@banata-auth/react/convex` | Convex provider bridge |

---

## Create The Browser Client

```ts
"use client";

import { createAuthClient } from "@banata-auth/react/plugins";

export const authClient = createAuthClient({
  baseURL: "/api/auth",
});
```

---

## Use The Prebuilt Forms

```tsx
"use client";

import Link from "next/link";
import { SignInForm, SignUpForm } from "@banata-auth/react";
import { authClient } from "@/lib/auth-client";

export function SignInScreen() {
  return (
    <SignInForm
      authClient={authClient}
      callbackURL="/"
      footer={<Link href="/sign-up">Create an account</Link>}
    />
  );
}

export function SignUpScreen() {
  return (
    <SignUpForm
      authClient={authClient}
      callbackURL="/"
      footer={<Link href="/sign-in">Already have an account?</Link>}
    />
  );
}
```

These forms call your app's `/api/auth` route. They do not connect to the Banata backend directly from the browser.

## Provider And Hooks

If you want auth state inside client components, wrap your app with `BanataAuthProvider` and provide an adapter:

```tsx
"use client";

import { BanataAuthProvider } from "@banata-auth/react";

const adapter = {
  fetchSession: async () => {
    const response = await fetch("/api/auth/get-session");
    if (!response.ok) return null;
    return response.json();
  },
  signOut: async () => {
    await fetch("/api/auth/sign-out", { method: "POST" });
  },
};

export function AuthProvider({ children }: { children: React.ReactNode }) {
  return <BanataAuthProvider adapter={adapter}>{children}</BanataAuthProvider>;
}
```

Then use:

- `useBanataAuth()`
- `useUser()`
- `useSession()`
- `useOrganization()`

## Organization Client

Banata re-exports an organization client wrapper that accepts your project's custom role slugs.

```ts
import {
  createAuthClient,
  organizationClient,
} from "@banata-auth/react/plugins";

export const authClient = createAuthClient({
  baseURL: "/api/auth",
  plugins: [organizationClient()],
});
```

This is important because Banata's RBAC model is `super_admin` first, with custom roles created inside the project. Customer apps should not be limited to a fixed `member | admin | owner` role set.

---

## Convex Bridge

If your frontend also talks to Convex directly, use `BanataConvexProvider` from `@banata-auth/react/convex` to bridge Banata auth tokens into your Convex client.

That is a frontend integration detail. It does not change the Banata project model.
