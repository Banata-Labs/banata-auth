# Dashboard go-live setup

This app is wired to its own Convex project and Better Auth backend.

## 1) GitHub OAuth app

Create or reuse a GitHub OAuth app and configure:

- Authorization callback URL:
  - `http://localhost:3001/api/auth/callback/github` (local)
  - `https://<your-domain>/api/auth/callback/github` (prod)

## 2) Required env vars (dashboard app)

Create `apps/dashboard/.env.local` with:

- `SITE_URL=http://localhost:3001`
- `BETTER_AUTH_SECRET=<strong-random-secret>`
- `GITHUB_CLIENT_ID=<github-oauth-client-id>`
- `GITHUB_CLIENT_SECRET=<github-oauth-client-secret>`

`CONVEX_DEPLOYMENT`, `NEXT_PUBLIC_CONVEX_URL`, and `NEXT_PUBLIC_CONVEX_SITE_URL`
are generated by `convex dev`.

## 3) Push env vars to Convex deployment

From `apps/dashboard`:

```bash
bunx convex env set SITE_URL "$SITE_URL"
bunx convex env set BETTER_AUTH_SECRET "$BETTER_AUTH_SECRET"
bunx convex env set GITHUB_CLIENT_ID "$GITHUB_CLIENT_ID"
bunx convex env set GITHUB_CLIENT_SECRET "$GITHUB_CLIENT_SECRET"
```

## 4) Start

From repo root:

```bash
bunx turbo run dev --filter=@banata-auth/dashboard
```

Open `http://localhost:3001/sign-in` and sign in with GitHub.
