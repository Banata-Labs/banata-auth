## 2026-03-09 External Consumer Journey

### Issues Found
- Install warning: npm-published Banata packages emit peer dependency warnings under Bun during a clean external install (`incorrect peer dependency better-auth`, `better-call`, `@better-auth/core`, `convex`). Need to verify published peer dependency ranges and package manager compatibility.
- Next.js prerender issue: a straightforward `createAuthClient({ baseURL: "/api/auth" })` setup fails during `next build` with `BetterAuthError: Invalid base URL: /api/auth`. External apps need a server-safe absolute fallback (`NEXT_PUBLIC_SITE_URL`) even though this is not obvious from the package surface.
- Fixed in source: the shared Next.js auth proxy was forwarding decompression headers (`content-encoding`, `content-length`, `transfer-encoding`) after Undici had already decompressed the upstream Better Auth response. This broke social sign-in in real browsers with `ERR_CONTENT_DECODING_FAILED` after `POST /api/auth/sign-in/social`. The fix now strips those transport headers before returning the proxied response.
- Hosted testing constraint: automated browsers are blocked by dashboard BotID protection on `POST /api/auth/sign-in/social` with `403 {"error":"Bot detected. Access denied."}`. Human browser testing still works, but full hosted E2E automation now needs either a documented manual checkpoint or an internal test bypass.
- Hosted dashboard issue: API key creation on `https://auth.banata.dev/api-keys` is still serving the stale auth runtime that rejects scoped key metadata with `400 {"code":"METADATA_IS_DISABLED","message":"Metadata is disabled."}`. That causes project-scoped API key creation to fail or create unusable unscoped keys. Local source has the fix, but the hosted deployment is not yet on that build.
- Follow-up on 2026-03-10:
  - confirmed root cause was a stale Convex prod deployment (`warmhearted-peccary-760`), not the Vercel frontend build
  - manually deployed the clean `origin/main` snapshot to Convex prod
  - next verification pass is blocked by hosted `POST /api/auth/sign-in/social` returning `403` before the GitHub redirect in automated headed-browser testing
- Local developer issue: `apps/dashboard` consumes built workspace packages, so running `bun run dev` without rebuilding `@banata-auth/convex`, `@banata-auth/react`, `@banata-auth/sdk`, `@banata-auth/shared`, and `@banata-auth/nextjs` can silently run stale auth behavior. Fixed locally by expanding the dashboard `predev` script to build all consumed packages first.
